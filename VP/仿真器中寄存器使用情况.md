VP_reg包含的寄存器在VP指令中都用到了吗？



不是。按当前代码看，`VP_reg` 里有些字段被 VP 指令真实读取，有些只是能被 `VP_SET*` 写入，但当前 VP compute 路径没有 consumer。

主要结论：

- `IQK0 / IQB0 / IQK1 / IQB1`：当前只在 `VP_SETV-*` 中写入，没有在 VP compute 路径中读取。`reg_config_tracker.py` 也明确写了：`VP.SETV_IQK0/IQB0/IQK1/IQB1` 没有 consumer。
- `PSMO`：在 `VP_SETM-SPM {PSMO,SCSQ}` 中会写入，但当前没有读者；同组的 `SCSQ` 会在 scalar sqrt 路径中使用。
- `QK / QB`：只有 `MD.QEN == 1` 时，在 `VP_post_cal()` 里用于输出量化：`result = QK * result + QB`。
- `STDW / STDH / KWS / KHS / IPW / IPH / PADV`：只在 compare-pool 或 sum-tree-pool 这类 kernel 路径里用，不是每条 VP 指令都会用。
- `SG0V / SG1V / MG0V / MULV / SUMV / CMPV / MSKV`：都是条件使用，取决于 `LUTG0/LUTS/GPMD/GPFS/SCMD/SRLC/CGPM` 等模式位。
- `MD / LUTG0 / LUTG1 / LUTS / CGPM / GPMD / SCMD / GPFS / GPFSMDL1 / SCFS / QMD / SRLC`：属于 VP 计算路径的模式控制类寄存器，很多字段用于选择模块、分支、后处理或数据通路。
- `result_register / param_gen_engine_* / shift_engine_b / param_gen_scalar / mask_info`：更像 VP executor 的内部中间状态/缓存，不是单纯 ISA 配置寄存器；这些在 VP 执行过程中会读写。

所以答案是：**不是 VP_reg 中所有寄存器都已在 VP 指令执行中用到；当前明确未被真实消费的至少包括 `IQK0/IQB0/IQK1/IQB1` 和 `PSMO`。其他很多字段也是条件使用，而不是每条 VP 指令都会读取。**
