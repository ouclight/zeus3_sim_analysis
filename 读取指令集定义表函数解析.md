`指令录入()` 的作用是：读取 `Zeus-v3指令格式v1.4.xls` 指令格式表，把每条指令的 64-bit 字段布局解析成一个 Python 字典返回。

位置：[指令解析.py (line 7)](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_arch/指令解析.py:7)

它最终返回的结构大致是：

```
{
    "某条指令助记符": {
        "inst_type": "指令类型",
        "字段名": {
            "length": 字段bit宽度,
            "index": [高bit位置, 低bit位置]
        },
        ...
    },
    ...
}
```

执行流程：

1. 找到 Excel 文件
   它基于当前脚本目录拼出：

```
Zeus-v3指令格式v1.4.xls
```

然后用 `xlrd.open_workbook()` 打开，读取第 2 个 sheet：

```
table = ori_book.sheets()[1]
```

1. 把 Excel 内容读成二维数组 `inst_xls`
   每一行对应一条指令格式，每一列对应一个字段/bit 单元。
2. 清理单元格类型
   如果 Excel 单元格是 `float`，转成 `int`。这是因为 Excel 读出来的数字常常是浮点，例如 `1.0`，但指令 bit 字段需要整数。
3. 补齐 A 列合并单元格
   Excel 里 A 列可能用了合并单元格，导致后续行为空。代码把空值补成上一行的值：

```
if inst_xls[row][0] == '':
    inst_xls[row][0] = inst_xls[row - 1][0]
```

1. 跳过中文说明行
   如果第 0 列包含中文，认为这是说明性行，不是正式指令格式，直接跳过。
2. 构造 `bit_list`
   从第 2 列开始扫描每个 bit 字段。非空单元格表示一个新字段，空单元格表示前一个字段继续占用 1 bit。

例如连续 3 个 bit 都属于 `opcode`，会被整理成：

```
["opcode", 3]
```

如果字段名是 `rsv`，会改成 `rsv_0`、`rsv_1`，避免字典 key 重名。

1. 合并连续数字字段和连续 `x` 字段
   有些固定 bit 值在 Excel 中可能按单 bit 写成 `1 0 1 0`，代码会把连续数字拼成一个字段，例如 `"1010"`，并累加长度。

连续的 `x` 也会合并，表示 don't care / unused bit 段。

1. 计算每个字段的 bit index
   默认一条指令是 64 bit，从 bit 63 开始往低位排：

```
cur_index = 63
```

每遇到一个字段，就记录：

```
"字段名": {
    "length": 字段宽度,
    "index": [高位, 低位]
}
```

然后 `cur_index` 往低位移动。

1. 用助记符作为 key 存入 `inst_dic`
   `inst_xls[row][1]` 是指令助记符，例如某条完整 instruction string。
   如果发现助记符重复，会打印该行。
2. 返回结果
      代码以前可能会写出 `inst.json`，但现在写文件逻辑被注释掉了：

```
# 不再写 inst.json 到 CWD, 避免多进程并行仿真时的文件竞态
```

所以当前函数只返回内存中的 `inst_dic`。

简而言之：`指令录入()` 是一个“Excel 指令格式表 -> 指令字段描述字典”的转换器，用来让仿真器知道每条 64-bit 指令中各字段的位置和长度。
