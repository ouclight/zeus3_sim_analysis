按代码看，`zeus_v3_vp_case_gen/zeusv3_asm` 里的汇编/反汇编是“表驱动”的：指令格式和 opcode 定义在 [src/isa/zeus_v3.py (line 1)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/src/isa/zeus_v3.py:1)，汇编器按这张表把 `FIELD[value]` 写入 64-bit 指令，反汇编器再按同一张表从 64-bit 指令中抽字段还原文本。

**汇编流程**

1. 命令行入口：[pb-asm.py (line 10)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/pb-asm.py:10)

   - 接收 `asm_file`、`--output/-o`、`--version/-v`。
   - 默认输出为输入同名 `.bin`。
   - 创建 `AssemblerWrapper`，调用 `assemble(asm_path)`。
   - 将每条机器码 `int` 写成 8 字节 little-endian 到 `.bin`。

2. API 包装层：[src/api.py (line 25)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/src/api.py:25)

   - ```
     AssemblerWrapper.assemble()
     ```

      支持三种输入语义：

     - 字符串且是文件路径：读取文件，每个非空行作为一条汇编。
     - 字符串但不是文件：当作单条汇编。
     - `list[str]`：当作多条汇编。

   - 对文件/list 输入，会先做 `_check_sync_validity()` 邮箱依赖检查。

3. 邮箱同步检查：[src/api.py (line 65)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/src/api.py:65)

   - 执行单元集合为 `VP, DT, PE, SO, ST, SI, LW, LD, CT`。
   - 扫描每条含 `RecvMails` 的指令。
   - `SendMails[A]` 会给 `当前unit -> A` 的 mailbox 计数加 1。
   - `RecvMails[A]` 会要求已有 `A -> 当前unit` 计数，并消耗 1。
   - 文件结束时要求所有 mailbox 计数归零，否则报错。
   - 注意：单条字符串输入不会做这个全局同步检查。

4. 真正编码：[src/asm/assembler.py (line 45)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/src/asm/assembler.py:45)

   - 取第一段作为 mnemonic，例如 `VP_STSR`、`CT_MOV_R_I`。
   - 在 `instr_format` 中查该 mnemonic。
   - 用正则解析所有 `字段名[内容]`，例如 `Rx[0]`、`RecvMails[VP,DT]`。
   - 对每个格式字段：
     - `opcode`：查 `opcode` 表，写入 bit `[63:56]`。
     - 二进制字段名如 `"0000"`：作为固定位写入。
     - 普通字段：取用户给的值；没给则默认 0。
     - `RecvMails/SendMails`：转成 9-bit bitmask。
   - 每个字段按 `length` 和 `index` 写入目标 bit 位。

邮箱 bit 顺序在 [src/asm/assembler.py (line 13)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/src/asm/assembler.py:13) 定义为：

```
bit0 VP, bit1 DT, bit2 PE, bit3 SO, bit4 ST, bit5 SI, bit6 LW, bit7 LD, bit8 CT
```

**反汇编流程**

1. 命令行入口：[pb-disasm.py (line 10)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/pb-disasm.py:10)
   - 接收 `inst_file`、`--output/-o`、`--version/-v`、`--hex/-x`。
   - 默认输出为输入同名 `.disasm`。
   - 创建 `DisassemblerWrapper`，调用 `disassemble(inst_path)`。
   - 如果带 `--hex`，输出前缀地址 `0x00000000:`，地址按每条 8 字节递增。
2. API 包装层：[src/api.py (line 146)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/src/api.py:146)
   - `int`：反汇编单条。
   - `list/tuple[int]`：反汇编多条。
   - 文件路径：按每 8 字节 little-endian 读成一个 64-bit `int`。
   - 如果最后不足 8 字节，会右侧补 0 后继续反汇编。
3. 真正解码：[src/disasm/disassembler.py (line 24)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/src/disasm/disassembler.py:24)
   - 从 bit `[63:56]` 提取 opcode。
   - 用反向 opcode 表找到 opcode 名。
   - 在所有 `instr_format` 中找 opcode 匹配、且固定位字段匹配的格式。
   - 当前实现选择第一个匹配格式。
   - 按字段 `length/index` 抽取字段值。
   - 跳过固定位字段，把普通字段输出成 `FIELD[value]`。
   - `RecvMails/SendMails` 会把 9-bit mask 还原为 `VP,DT,...` 字符串。

**Python 指令构造类**
`src/isa/*_instr.py` 不是二进制编解码核心，但它们负责用 Python 对象生成合法汇编字符串。例如 [src/isa/lw_instr.py (line 8)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/src/isa/lw_instr.py:8) 的 `LW_MOV.asm()` 返回：

```
LW_MOV BRx[...] BRa[...] WBS_IE[...] WBS[...] RecvMails[...] SendMails[...]
```

这些字符串再交给 `AssemblerWrapper` 走同一条汇编路径。`BaseInstruction` 还提供 `add_prev/add_next` 来拼 `RecvMails/SendMails`，见 [src/isa/base.py (line 11)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/src/isa/base.py:11)。`Function` 可以收集这些指令对象并生成 asm list，见 [src/core/function.py (line 27)](/home/zhangds/Zeus3FunctionalSimulator/zeus_v3_vp_case_gen/zeusv3_asm/src/core/function.py:27)。

一个需要注意的点：虽然 CLI/API 都有 `--version` / `zeus_version` 参数，但当前 `Assembler` 和 `Disassembler` 实际都是静态 import `src/isa/zeus_v3.py`，版本参数只被保存，没有真正切换 ISA 表。
