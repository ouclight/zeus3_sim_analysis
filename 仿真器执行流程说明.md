下面按 `main_debug.py` 入口梳理工程主流程。

**整体流程**
1. `main_debug.py` 解析参数：case 路径、运行模式、timing/cost、DT/VP 执行器数量、寄存器配置检查、compare 开关等。入口默认 case 是 `test/debug/ct_sim/ct_jump_jz_-428451268_-428449617_29`，默认模式是 `parallel-pc`。见 [main_debug.py](/home/zhangds/Zeus3FunctionalSimulator/main_debug.py:27)。
2. 校验 case 目录和 `init.json`，读取 `simu_params`。同时调用 `指令录入()` 从 `Zeus-v3指令格式v1.4.xls` 生成指令字段解析字典。见 [main_debug.py](/home/zhangds/Zeus3FunctionalSimulator/main_debug.py:87) 和 [指令解析.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_arch/指令解析.py:7)。
3. 创建 `NPUSimulator(inst_json, simu_params)`，把 DT/VP 实例数、timing 配置、寄存器检查开关注入 `simu_params`。见 [main_debug.py](/home/zhangds/Zeus3FunctionalSimulator/main_debug.py:116)。
4. 根据 `--mode` 选择执行：
   - `serial` -> `NPUSimulator.run()`
   - `parallel` -> `NPUSimulator.run_parallel()`
   - `parallel-pc` -> `NPUSimulator.run_parallel_PC()`，这是默认路径。见 [main_debug.py](/home/zhangds/Zeus3FunctionalSimulator/main_debug.py:78)。
5. 仿真结束后转储 trace 和 GDG 输出，再做 golden compare；若启用 `--check-reg-config`，额外输出无效寄存器配置报告。见 [main_debug.py](/home/zhangds/Zeus3FunctionalSimulator/main_debug.py:143)。

**初始化阶段**
`NPUSimulator` 是多核顶层对象。它根据 `init.json` 里的 `core_map` 创建 die/core 拓扑，并初始化两类全局/核内存：

- GDG：每个 die 一个 `MemoryManager`，包含两个 GDG DDR block，base 分别是 `0x10F_0000_0000` 和 `0x11F_0000_0000`。见 [simulator.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/simulator.py:57)。
- WL1/weight：每个 core 独立一个 `MemoryManager`，base 为 `0x100_0000_0000 + core_id * 0x10_0000_0000`。见 [simulator.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/simulator.py:68)。
- 通过 `GDG_init_info` 和 `weight_init_info` 把 case 里的二进制文件加载到对应内存。见 [simulator.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/simulator.py:78)。

随后为每个 die/core 创建 `NPUCore`。`NPUCore` 内部组装了本 core 的 L2、寄存器、执行器、trace、timing 和调度器。见 [core.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/core.py:27)。

**Core 内部组成**
每个 `NPUCore` 主要包含：

- `Regfile`：通用寄存器 `Rx/ARx/FRx`、PC、condition code、RR/BR/VR/VP/PE 寄存器、SI/SO 同步信息等。见 [regfile.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/regs/regfile.py:5)。
- `L2Buffer`：本 core 的 L2 SRAM 模拟，支持按 BR/RR 读写、滚筒 buffer 地址映射、tile 读写。见 [L2_buffer.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/memory/L2_buffer.py:4)。
- 执行器：
  - `CT_excuter_class`
  - `LD_excuter_class`
  - `ST_excuter_class`
  - `DT_excuter_class`，支持多实例
  - `VP_excuter_class`，支持多实例
  - `PE_excuter_class`
  - `SI_excuter_class`
  - `SO_excuter_class`
- `instruction_dispatch_class`：负责加载指令、解析字段、维护 mailbox、按类型分发到执行器队列。见 [core.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/core.py:55)。

**指令加载与解析**
调度器初始化时会从 GDG 中读取指令，而不是直接从文件读。入口地址来自 `init.json` 的 `core_map.INSTRUCTION_ADDR`，每条指令 8 字节，小端读取，直到遇到 `CT_ED` 停止。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:1195)。

指令类型由 `opcode_518.parse_instruction_type()` 解析；字段解析规则来自 `指令录入()` 生成的 `inst_dic`。字段解析时会按 xls 里的 bit index/length 提取值，并把 `Send Mails(9bit)` / `Rec Mails(9bit)` 展开成对各执行单元的 mailbox bit。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:1209)。

**默认执行路径：parallel-pc**
`main_debug.py` 默认 `--mode parallel-pc`，所以会调用 `NPUSimulator.run_parallel_PC()`：为每个 core 启动一个 `dispatch_thread_PC`，然后 join 等待所有 core 结束。见 [simulator.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/simulator.py:433)。

`instruction_dispatch_PC()` 的核心逻辑是：

- 使用 `core_regs.PC` 作为真实 PC，在 while 循环中取当前指令。CT jump 可以修改 PC，因此适合调试控制流。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:235)。
- `CT*` 指令同步执行，直接调用 CT 执行器；`CT_ED` 结束本 core 调度。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:248)。
- `VP_SET*`、`PE_SETM` 这类配置指令也直接执行，保证配置写入与后续消费的顺序清晰。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:278)。
- LD/ST/DT/VP/PE/SI/SO 的计算或搬运指令通常会先 latch 相关 BR/VP/PE 寄存器快照，再入队给对应 worker。这样模拟“指令发射时锁存寄存器”的行为。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:325)。
- 循环末尾普通情况下 `PC += 1`，并记录 PC trace。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:427)。
- 所有指令分发完后等待各队列 `join()`，再保存 core/executor trace。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:441)。

**并行调度与 mailbox**
调度器为 LD/DT/ST/VP/PE/SI/SO 建立主队列和 worker 线程。DT/VP 还额外有多实例执行器队列，用 `ExecutorRouter` 根据 `CSS(CalSrcSel)` 路由。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:61)。

mailbox 是一个二维 token 表：`mailbox[目标单元][来源单元]`。执行前检查 `Rec Mails` 是否满足，执行结束后按 `Send Mails` 给下游投递 token。等待超时会抛出带 PC、等待项、当前 mailbox 状态的死锁错误。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:1265)。

DT/VP 的 worker 比较特殊：

- `DT_MAIL` / `VP_MAIL` 在主 worker 内直接处理，避免 mail-only 指令和真实 compute 指令在多实例队列里乱序。
- `DT_TRA/DT_RES` 和 `VP_STSR/VP_DTSR` 会根据 CSS 路由到具体 DT/VP 实例队列。
- 同时做简单 L2 区域冲突检测。见 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:964) 和 [instruction_dispatch.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_dispatch/instruction_dispatch.py:1045)。

**执行器职责**
- `CT_excuter_class`：控制/标量寄存器/BR/VR/PC 类操作，包括 `MOV/FMOV/VMOV/BMOVM/SET*/CALC/logic/JUMP/WAIT/SEND/MAIL`。CT 是控制流核心，`CT_JUMP` 会影响 PC，`CT_WAIT/SEND` 处理跨核同步。见 [CT_excuter_class.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_arch/CT_excuter_class.py:20)。
- `LD_excuter_class`：从 GDG 或 weight DRAM 读 tile，写入 L2。`LD_MOV` 根据 BR 描述和 AR 页偏移计算源地址。见 [LD_excuter_class.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_arch/LD_excuter_class.py:12)。
- `ST_excuter_class`：从 L2 读 tile，按 BR 描述写回 GDG 或 weight DRAM。见 [ST_excuter_class.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_arch/ST_excuter_class.py:3)。
- `DT_excuter_class`：数据重排/transpose，典型是 `DT_TRA`，从 L2 读 BR tile，按 order 转置，再写回 L2。见 [DT_excuter_class.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_arch/DT_excuter_class.py:9)。
- `VP_excuter_class`：向量处理单元，支持 `VP_SET*` 配置、`VP_STSR/VP_DTSR` 计算，以及 compare、sum、mul、floor、shift、relu、topk、scalar 等子模块。见 [VP_excuter_class.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_arch/VP_excuter_class.py:42)。
- `PE_excuter_class`：矩阵/卷积类计算，核心路径是 `PE_CONV_excuter_FAST`，从 L2 读 activation、从 weight DRAM 读权重，支持 KVCache/index table 等权重布局逻辑。见 [PE_excuter_class.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_arch/PE_excuter_class.py:64)。
- `SI_excuter_class` / `SO_excuter_class`：跨 core 输入/输出同步与数据搬运，使用 `SI_syncinfo` / `SO_syncinfo` 做 pairing，MOV 时通过对端 core 的 syncinfo 传递 L2 tile 数据。见 [SI_excuter_class.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_arch/SI_excuter_class.py:3) 和 [SO_excuter_class.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/instruction_arch/SO_excuter_class.py:3)。

**收尾输出**
仿真结束后：

- `dump_memory_and_save_trace()` 保存 NPU 级 trace 到 `log_npu_level.json`，并按 `GDG_dump_info` 把 GDG 指定区域 dump 到如 `simu_out0.bin`。见 [simulator.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/simulator.py:138)。
- `run_compare()` 从 `GDG_dump_info.D0.dump_0.DF` 取输出数据类型，与 case 下 `output.bin` 比较。浮点类走 FP32 解码后余弦相似度，整数类走严格二进制比较。见 [cli_helpers.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/cli_helpers.py:35) 和 [simulator.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/simulator.py:324)。
- 若启用 `--check-reg-config`，`RegConfigTracker` 会输出每 core 和汇总的无效寄存器配置报告。见 [simulator.py](/home/zhangds/Zeus3FunctionalSimulator/npu_core_simulator/simulator.py:478)。

简化调用链可以理解为：

```text
main_debug.py
  -> parse_args / load init.json / 指令录入
  -> NPUSimulator
      -> MemoryManager 初始化 GDG/WL1
      -> NPUCore[*]
          -> Regfile / L2Buffer / executors / instruction_dispatch_class
              -> load_instruction_from_GDG
              -> start worker threads
  -> run_parallel_PC
      -> per-core dispatch_thread_PC
          -> parse instruction fields
          -> CT/config 直接执行
          -> compute/move 指令 latch 后入队
          -> worker 等 mailbox -> executor 执行 -> send mailbox
  -> dump_memory_and_save_trace
  -> run_compare
  -> optional write_reg_config_reports
```
